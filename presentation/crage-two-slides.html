Website design — Crage

What you see

  • Phone-style shell — single-column layout (full width on small screens) with a chat header, scrollable thread, and composer.
  • Animated face — SVG clock in the header reflects the current state (neutral, surprised, anxious, etc.) as people text.
  • iMessage-like chat — user messages on the right; Crage’s replies on the left, with a short typing indicator before each reply.
  • Live sync — the page calls POST /message to send text and GET /state every couple of seconds so time-based changes (surprise fading, idle) update without reloading.

Under the hood (stack)

  • Flask serves HTML, CSS, JS, and the JSON API.
  • One shared session on the server: everyone texting the deployed link shares the same conversation “room” (in-memory ClockBrain).

What the LLM does (and doesn’t)

Single job for the LLM — classification only

  • For each incoming message (plus recent history), the model in emotion.py chooses one label: roughly how the attendee sounds — neutral, joyful, first-time problem (anxious), repeat complaint (embarrassed), or strong frustration / yell / third strike (angry).
  • It may also emit a short topic (e.g. captions, audio) so the brain can detect “same problem again.”
  • If no API key is set, a heuristic (regex + rules) does the same job instead — same contract, no LLM.

Not an LLM task — the actual reply text

  • Chat lines like “I’m sorry that’s happening…” come from fixed pools in brain.py (_REACTIONS), picked at random per state. That keeps voice consistent and avoids an extra model call per message.
  • brain.py also owns surprised flash timing and at_ease after idle — not the LLM.
User message → classifier (LLM or heuristic) → state + topic → ClockBrain picks reaction text + timestamps / surprise / idle → JSON to web + hardware
